# VOI 3D可视化与裁剪系统 - 实施说明文档

## 一、核心架构概述

本系统旨在将1000-2000张二值化TIFF切片在Web端重建为可交互3D模型，并实现三维空间裁剪。采用“**后端智能预处理 + 前端渐进加载**”架构，在保证5秒内可交互的前提下，通过可选扩展逐步提升模型质量。

## 二、数据传输方案（优化版）

### **阶段1：即时预览（必做核心）**
*   **目标**：5秒内显示可交互的3D模型概貌。
*   **后端处理**：
    1.  **智能抽帧**：采用`5帧抽1帧`策略（如1000帧→200帧）。
    2.  **空间降采样**：使用`Pillow`的`LANCZOS`算法，将切片分辨率从`512×512`降至`256×256`。（暂不使用）
    3.  **高效压缩**：对二值化数据（0/1）进行**行程编码（RLE）**，再使用`zipfile`打包。
*   **传输数据**：一个经过RLE和ZIP双重压缩的数据包，大小约 **2-3 MB**。
*   **前端接收**：一个包含约200张`256×256`二值化TIFF切片的ZIP文件。

### **阶段2：增强细节（推荐扩展）**
*   **目标**：在预览基础上，后台静默加载剩余数据，提升模型连贯性与细节。
*   **后端处理**：
    1.  **增量准备**：将未被阶段1抽取的剩余切片（约800帧），同样降采样至`256×256`。
    2.  **分块**：按Z轴顺序分成若干数据块（如每100帧一块）。
*   **传输数据**：多个数据块，总大小约 **8-10 MB**。前端在预览加载完成后，自动在后台请求。
*   **前端整合**：将新收到的数据块插入到现有体素网格的对应位置，动态更新3D模型。

## 三、完整工作流程

### **第一步：后端数据预处理与传输**
1.  **读取**：后端（Python Flask）使用`Pillow`库读取指定项目的原始TIFF序列。
2.  **处理**：
    *   **预览数据**：执行5抽1降采样，生成低分辨率序列。
    *   **增强数据**：准备剩余切片的降采样版本并分块。
3.  **压缩与响应**：对处理后的二值化数据应用RLE算法，打包为ZIP，通过`/api/voi/preview`等API接口发送给前端。

### **第二步：前端数据接收与3D重建**
1.  **请求与解压**：前端（Vue）通过`axios`请求数据，使用`pako`解压gzip流，再用`JSZip`解包ZIP文件。
2.  **解析与构建体素网格**：
    *   使用`utif.js`将每个TIFF切片解析为二值化像素数组（0或1）。
    *   将所有切片数据按顺序堆叠，在内存中构建成一个三维的**体素网格（Voxel Grid）**，用`Uint8Array`表示。
3.  **生成3D表面模型**：
    *   **核心算法**：调用 `isosurface` 库中的 **`surfaceNets`算法**（Marching Cubes的高效变种）。
    *   **过程**：该算法遍历体素网格，在值为0和1的体素边界处，根据预设阈值（如0.5）生成三角面片（顶点和索引）。
4.  **渲染与呈现**：将算法输出的顶点和索引数据，转换为`Three.js`的`BufferGeometry`，附上材质（颜色、透明度），创建为`Mesh`对象加入场景。配合`OrbitControls`实现旋转、缩放和平移。

### **第三步：交互式三维裁剪**
1.  **启用裁剪模式**：用户点击按钮后，系统在3D场景中创建一个半透明的红色线框立方体（`BoxGeometry`）作为裁剪框。
2.  **交互调整**：为裁剪框绑定`TransformControls`控件，用户可通过鼠标直接拖拽、缩放、旋转该框，直观地框选目标区域。
3.  **坐标计算与发送**：
    *   **核心转换**：实时将裁剪框在**3D世界坐标系**中的包围盒范围，转换为相对于体素网格原点的**体素索引坐标** `[x_min, x_max, y_min, y_max, z_min, z_max]`。
    *   **发起请求**：用户确认后，前端将这些索引坐标通过`POST /api/voi/crop`接口发送给后端。

### **第四步：后端精确裁剪与保存**
1.  **接收与解析**：后端接收裁剪坐标，验证其在原始数据尺寸范围内的合法性。
2.  **执行裁剪**：
    *   根据`z_min`和`z_max`确定需要处理的原始TIFF切片范围。
    *   对范围内每一张原始分辨率（`512×512`）的TIFF，使用`Pillow`或`numpy`进行二维数组裁剪（`img_array[y_min:y_max, x_min:x_max]`）。
3.  **保存结果**：将裁剪出的每个二维数组，保存为新的TIFF文件序列，存储在服务器指定路径。

## 四、技术栈与工具对应关系
| 环节 | 任务 | 使用工具/算法 |

| | 提供Web API | Flask |
| | 数据压缩 | RLE算法, zipfile, gzip |
| **前端渲染** | 3D渲染引擎 | Three.js |

## 五、关键设计要点
1.  **坐标系统一**：确保前后端对体素网格的**原点、坐标轴方向、空间间距**有完全相同的定义，这是裁剪坐标转换正确的基石。
2.  **数据保真分离**：预览模型的**视觉质量**（由抽帧和降采样决定）与最终裁剪结果的**数据精度**（始终使用原始TIFF）是解耦的。前者保证交互流畅，后者保证结果准确。
3.  **渐进式体验**：系统优先保障“快速可见可操作”，通过后台静默加载、按需加载等方式平滑提升画质，避免用户长时间等待。
4.  **性能边界**：前端体素网格的大小（约`256x256x200`）和Marching Cubes算法的计算量，需在主流浏览器承受范围内。扩展阶段的分块加载是保持流畅的关键。
