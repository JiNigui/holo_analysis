# 孔洞分析软件项目架构与功能详解
## 前端架构
### 技术栈
- 框架 : Vue.js 2.6.10
- UI组件库 : Element UI 2.15.14
- 路由管理 : Vue Router 3.0.6
- 状态管理 : Vuex 3.1.0
- HTTP请求 : Axios 1.13.1
- 基础框架 : vue-admin-template 4.4.0
- 构建工具 : Vue CLI 4.4.4
- 模拟数据 : Mock.js
- 样式处理 : SCSS

### 主要功能模块
1. 用户认证模块
   
   - 登录、登出功能
   - 个人信息管理
   - 基于JWT的认证机制
2. 项目管理模块
   
   - 项目列表展示
   - 创建新项目
   - 项目详情查看
   - 项目编辑和删除
3. 管理员功能模块
   
   - 用户管理（添加、编辑、删除用户）
   - 系统日志查看
   - 权限分配
4. 孔洞分析模块
   
   - 文件上传界面
   - 分析任务管理
   - 分析结果可视化展示
### 权限控制
采用基于角色的权限控制（RBAC），分为：

- 普通用户（user）
- 管理员（admin）
路由配置中通过meta.roles字段控制页面访问权限，确保用户只能访问有权限的功能。

### 核心功能实现
1. 用户管理功能
   
   - 用户注册、登录认证
   - JWT Token生成与验证
   - 用户信息管理
   - 操作日志记录
2. 项目管理功能
   
   - 项目的增删改查操作
   - 基于用户权限的项目访问控制
   - 项目状态管理
3. 孔洞分析功能
   
   - 图像文件上传处理
   - 调用分析引擎执行6步分析流程
   - 分析结果存储与获取
   - 特征数据（孔洞面积、周长、宽高比等）处理
4. 权限管理
   
   - 用户角色划分（普通用户/管理员）
   - 基于角色的API访问控制
   - 操作日志记录与审计
### 数据库设计
主要数据模型包括：

- User : 用户信息
- Project : 项目信息
- Feature : 孔洞特征数据
- OperationLog : 操作日志记录
### 分析引擎集成
后端通过analysis_service.py调用hole-analysis目录下的6个分析步骤，实现完整的孔洞分析流程：

1. 原始图像二值化
2. 感兴趣区域选择
3. Mask R-CNN孔洞识别
4. 数据预处理
5. 目标孔洞寻找与切片
6. 形态学分析
## 前后端交互流程
1. 用户认证流程
   
   - 前端发送登录请求
   - 后端验证凭证，返回JWT Token
   - 前端存储Token，后续请求携带Token进行认证
2. 项目管理流程
   
   - 创建项目：前端填写表单 → 后端验证并存储 → 返回项目信息
   - 项目列表：前端请求 → 后端根据权限返回项目列表 → 前端展示
   - 项目详情：前端请求特定项目 → 后端验证权限并返回详情
3. 孔洞分析流程
   
   - 上传图像：前端选择文件 → 后端保存并关联项目
   - 开始分析：前端发起分析请求 → 后端调用分析引擎 → 返回任务ID
   - 获取结果：前端轮询结果 → 后端返回分析数据 → 前端展示可视化结果
## 系统特点
1. 模块化架构 ：前后端均采用模块化设计，便于维护和扩展
2. 分层设计 ：后端采用API层、Service层、Model层的分层架构
3. 权限控制 ：完善的基于角色的权限管理机制
4. 分析引擎 ：集成专业的孔洞分析算法，支持完整的6步分析流程
5. 前后端分离 ：采用主流的前后端分离架构，便于开发和部署
